# 買い物価格記録

店舗ごとの商品価格を記録し、商品ごとの最安値や単価を確認するための静的Webアプリです。
ログインやサーバーは不要で、GitHub Pagesにそのまま公開できます。

※ アプリ名は現在仮称です。
今後の開発状況に応じて変更する場合があります。

## 主な機能

- 商品名・店舗名・価格だけで素早く記録
- 未登録の商品や店舗は保存時に自動追加
- 商品カテゴリの自動推測
- 内容量(g)・個数から、gあたり/個あたりの単価を計算
- 商品ごとの最安値、最安店舗、最新価格、平均価格、過去最安値を表示
- 価格推移グラフ、店舗比較グラフ
- 商品、店舗、価格の詳細編集
- CSVエクスポート/インポート
- Open Food Factsの商品候補検索
- 近くのお店検索
- バーコード手入力検索、対応ブラウザでのバーコード読み取り

## 使い方

1. `index.html` をブラウザで開きます。
2. 「登録」タブで商品名、店舗名、価格を入力して保存します。
3. 必要な場合だけ「詳細設定」を開き、メーカー、カテゴリ、バーコード、内容量、個数、メモを入力します。
4. 「最安値」や「グラフ」で価格を確認します。
5. 「商品」「店舗」「価格」タブで登録済みデータをあとから編集できます。
6. 「データ」タブからCSVでバックアップできます。

価格、内容量、個数は数値だけを入力します。
画面上では `円`、`g`、`個` の単位が自動表示されますが、保存データやCSVには数値のみが入ります。

## Open Food Factsとバーコード

Open Food Factsの商品データは参考情報として使用します。
価格は取得せず、店舗ごとの価格はユーザーが入力します。

カメラ読み取りは、対応ブラウザではBarcodeDetector APIを使用します。
BarcodeDetector APIが利用できない場合は、ZXing (`@zxing/library 0.21.3`)を使用します。

読み取り対象は、原則としてEAN-13、EAN-8、UPC-Aです。
カメラでは中央の読み取り枠だけを解析し、有効なチェックデジットを持つ同じ番号を短時間に3回連続で検出した場合だけ読み取りを確定します。

読み取り後は番号の確認画面を表示します。
「この番号で検索する」を押すまで商品検索や保存は行いません。

カメラで読み取れない場合は、バーコード番号を直接入力して検索するか、商品名を手入力して商品候補を検索できます。

iPhone Safariを含め、カメラやライト、ズームの対応状況は端末とブラウザによって異なります。
カメラはHTTPSで公開されたページ、またはlocalhostで利用してください。
カメラが起動しない場合や読み取れない場合は、バーコード番号または商品名を手入力してください。

店舗独自のバーコードや値引きシールは、商品情報を取得できない場合があります。

### 商品検索の順番

1. localStorageに保存済みのJANコード一致商品
2. Open Food Factsのバーコード完全一致検索
3. 見つからない場合は手入力

Yahoo!ショッピング商品検索APIは日本の商品検索に有効ですが、Client IDが必要です。
楽天市場商品検索APIはApplication IDとAccess Keyが必要で、EC出品単位の検索となります。

GitHub PagesのHTMLやJavaScriptに認証情報を直接埋め込むと公開されるため、現在のプレビュー版ではYahoo!ショッピングと楽天APIへ接続していません。
将来追加する場合は、Cloudflare Workers、Netlify Functions、Vercel Functionsなどを経由し、APIキーをサーバー側の環境変数で管理します。

商品候補を選択して保存すると、JANコード、商品名、メーカー、カテゴリ、内容量、内容量単位、画像URL、情報の取得元、最終更新日を保存します。
以前のバージョンで保存した商品には、起動時に不足項目を空欄で補います。

## 近くのお店検索

位置情報は「近くのお店を探す」ボタンを押した時だけ使用します。
取得した緯度・経度は保存しません。

住所検索は全国の住所・地名に対応しています。
入力例は `○○駅 / ○○市` のような汎用表現です。
住所検索にはOpenStreetMap / Nominatimを使用します。
入力した住所や地名は位置検索のためだけに使用し、アプリ内には保存しません。

店舗候補にはOpenStreetMapのデータを使用します。
© OpenStreetMap contributors.
店舗情報は参考情報であり、実際の店舗名、場所、営業状況などと異なる場合があります。

住所・駅名・市区町村の検索にはNominatimを使用し、取得した地点を店舗検索の中心にします。
店舗や施設の検索にはOverpass APIを使用します。住所検索だけで店舗候補を作成することはありません。

現在地または指定地点の周辺を最初に3km以内で検索し、候補が少ない場合は5km、10kmの順に広げます。
候補は検索中心からの距離順に表示します。

店舗を選択して価格を記録すると、店舗名、住所、緯度、経度、OpenStreetMapの要素ID、店舗種別、取得元、最終利用日、利用回数を保存します。
次回は利用回数と最終利用日をもとに「最近使った店舗」を先に表示します。

重複店舗は、外部店舗ID、座標が約50m以内、店舗名と住所の一致、の順で判定します。

Yahoo!ローカルサーチAPIは日本のPOI検索に適していますが、Client IDが必要なため現在の静的版には組み込んでいません。
導入時は商品検索APIと同様にサーバーレス関数を経由します。

現在のバージョンには地図表示と地図上での位置指定はありません。
一覧からは店舗を選択でき、地図を追加する場合はLeafletとOpenStreetMapタイルを候補とします。

## 保存先

入力データは利用中のブラウザの `localStorage` に保存されます。
別の端末や別のブラウザには自動同期されません。

ブラウザの履歴・サイトデータを削除すると、保存したデータも消える場合があります。
機種変更、ブラウザ変更、キャッシュ削除の前には、必ず「データ」タブからCSVエクスポートでバックアップしてください。

保存形式のキーは従来と同じ `shoppingPriceTracker.v1` です。
商品・店舗に追加された項目は起動時に自動補完され、既存の商品、店舗、価格記録は削除されません。

## デザイン

白と淡いグレーを基調に、余白を広めにしたミニマルな画面にしています。
登録画面は基本3項目を優先し、詳細項目は折りたたみ式にして、スマホで素早く入力しやすい構成にしています。
